I’m currently building in the Replit Buildathon with Replit Agent 4, and my project is called [RoomForge](https://buildathons.replit.app/projects/5bc3942f-af85-40de-9b91-8eb4e1c07c3c).

This post is a simple update on what the event is, how the format works, and where my build is at right now.

## What the Buildathon is

The event runs for **3 weeks**.

The structure is straightforward: build in public, ship progress each week, and move from rough concept to a more complete product by final submission.

What I like about this format is the pacing. Three weeks is short enough to force focus, but long enough to iterate beyond a demo that only works once.

## Format and basic rules (practical version)

From a builder’s perspective, the process is mostly about consistency:

- register a project
- build and share progress during the event window
- submit updates on the weekly cadence
- ship a final version by the deadline

That sounds obvious, but it changes behavior. Weekly checkpoints create real pressure to make decisions instead of endlessly polishing.

## What I’m building: RoomForge

[RoomForge](https://buildathons.replit.app/projects/5bc3942f-af85-40de-9b91-8eb4e1c07c3c) is my project for this Buildathon.

The core idea is simple: help people design and iterate on room layouts faster, with AI as a practical collaborator instead of a novelty layer.

I’m aiming for a tool that helps with:

- exploring layout options quickly
- comparing tradeoffs between different room setups
- refining ideas in small iterations instead of big, one-shot plans

I’ll share more technical details as the build stabilizes, but the near-term focus is making the workflow useful end to end.

That part is never glamorous, but it matters. In short build cycles, a tight scope is usually the difference between a messy prototype and something people can actually try.

## Why this format works for me

A lot of AI building events can drift into vague demos. This one pushes toward concrete output because the timeline is visible and the checkpoints are frequent.

The best part so far is momentum. I’m not waiting for a perfect architecture doc. I’m shipping, checking what holds up, and adjusting week by week.

The grounded takeaway: a 3-week window is enough time to learn a lot if you treat each week as a real delivery deadline, not a draft.
